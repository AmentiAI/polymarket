## System Architecture

### 50-Point DNA Classification

Each candle gets a unique fingerprint across **15 dimensions**:

1. **Color** — GREEN / RED
2. **Shape** — HAMMER / DOJI / SHOOTING_STAR / STRONG / NORMAL
3. **Size** — TINY / SMALL / MEDIUM / LARGE (ATR-relative)
4. **Body %** — Percentage of total range
5. **Upper Wick %** — Percentage of total range
6. **Lower Wick %** — Percentage of total range
7. **RSI** — OB (>70) / OS (<30) / RISING / FALLING / MID
8. **MACD** — POS_STRONG / POS_WEAK / NEG_WEAK / NEG_STRONG
9. **BB Position** — ABOVE / NEAR_UPPER / MID / NEAR_LOWER / BELOW
10. **Volume** — HIGH / NORMAL / LOW (vs SMA)
11. **Trend** — UP / DOWN / FLAT (EMA fast vs slow)
12. **Streak** — 3G / 4G / 5G+ / 3R / 4R / 5R+ / MIXED
13. **Previous Color** — GREEN / RED
14. **2-Candle Pattern** — GG / GR / RG / RR
15. **3-Candle Pattern** — GGG / GGR / ... / RRR

**Train/Test Split**: 70/30 for validated accuracy (not overfitted)

### 24-Signal Weighted Voting System

**Core Signals Implemented** (weight in parentheses):

1. **DNA Prediction** (0.5) — Historical pattern matching
2. **2-Candle Pattern** (2.75) — GG/GR/RG/RR win rates
3. **3-Candle Pattern** (5.0) — GGG/GGR/.../RRR statistics
4. **4-Candle Pattern** (5.0) — 16 combo analysis
5. **Momentum** (1.5) — 5-candle price change
6. **Dildo Candle** (1.5) — Strong directional moves (body >80%, >1.5x ATR)
7. **Streak-5** (2.0) — Contrarian after 5+ same-color
8. **MACDstr** (2.0) — Strong MACD histogram moves

**Additional 16 signals**: RSI divergence, volume spikes, S/R proximity, sequences (1m/5m/15m), etc.

**Aggregation**: 
- Up score = Σ(weight × edge/100) for all UP signals
- Down score = Σ(weight × edge/100) for all DOWN signals
- Confidence = winning_score / total_score × 100

### Advanced Features

#### 1. Toxic Combo Hijacking ⚠️

**Problem**: Sometimes multiple signals all agree but historically fail.

**Solution**: Auto-detect these "toxic combos" from backtest data:
- Identify when 2+ signals all agree on direction
- Track verdict win rate when combo appears
- If WR <40%, flag as TOXIC
- **Live behavior**: Auto-flip verdict when toxic combo detected

**Example**:
```
Signals: RSI + MACD + Patterns all say UP (75% confidence)
Backtest: This combo historically wins only 35%
Action: FLIP to DOWN + log hijack reason
Result: Avoided trap, correct prediction
```

#### 2. RSI Zone Direction Flipping 🔄

**7 Multi-Timeframe Rules** that override predictions:

1. 5m RSI >73 + rising → Block GREEN (58.1% trap)
2. 15m RSI <32 + 5m falling → Flip to GREEN (bullish divergence)
3. 5m RSI >70 + 15m >65 → Block GREEN (double overbought)
4. 1m RSI <25 + 5m <40 → Force GREEN (extreme oversold)
5. 15m RSI >75 + 5m rising → Flip to RED (extended rally)
6. 5m RSI 30-40 + 15m <35 → Force GREEN (MTF oversold)
7. 1m RSI >80 → Block GREEN (parabolic exhaustion)

#### 3. MACD Red Zone Gating 🚫

**Rule**: When MACD histogram is falling outside ±6 deadzone → block GREEN bets

**Reason**: Backtests showed 58.1% win rate for GREEN in this zone = losing strategy

**Detection**: 
- Check current MACD vs previous bar
- If `hist_curr < hist_prev` AND `abs(hist_curr) > 6` → RED ZONE
- Shift left by 1 bar (if next is falling, flag current too)

#### 4. Backtest WR Lookup Table 📊

**Concept**: Only trade scenarios with proven historical performance

**Implementation**:
1. Run backtest on 30% test set (untouched during training)
2. Track WR for every `(direction, confidence_bucket)` pair
   - Buckets: 50-55%, 55-60%, 60-65%, 65-70%, 70%+
3. Build lookup: `{("GREEN", 60): {"wr": 72.3%, "n": 15}}`
4. **Live gate**: Only trade if backtested WR ≥60% AND n ≥5

**Example**:
```
Verdict: GREEN 62% confidence
Bucket: 60-65%
Backtest WR: 57.8% (12 samples)
Decision: SKIP (below 60% threshold)
```

#### 5. Relief Take-Profit 💰

**Scenario**: Position dips to -50%, looks dead, but recovers to +5%

**Normal**: Hold for resolution (might go to -100%)  
**Relief**: **SELL IMMEDIATELY** at any profit >0%

**Why**: Recovering from -50% is statistically significant. Lock in the survival win.

#### 6. Auto-Threshold Adjustment 🎚️

**Goal**: Maintain ~50% trade rate (avoid over/under-trading)

**How**:
- Every 120 candles (~10 hours): check trade rate
- If <45%: Lower thresholds by 3% (RSI, MACD, sequence, confluence)
- If >55%: Raise thresholds by 3%
- Cap at min/max safe ranges

**Effect**: Self-balancing system stays in optimal frequency zone
